+ 自動運転
+ ロボティクス

## 入門ガイド

デモンストレーションにはreComputer Robotics J4012を使用します。必要なものは以下の通りです：

+ SENSING GMSL2 カメラ
+ [Mini-Fakra ケーブル](https://www.seeedstudio.com/Mini-fakra-Coaxial-Cable-4-in-1-0-5m-Female-to-Female-p-6484.html)
+ [reComputer Robotics GMSL2 拡張ボード](https://www.seeedstudio.com/reComputer-Robotics-GMSL-board-p-6511.html)

これらの4つのチャンネルはデバイス上で**'A'**、**'B'**、**'C'**、**'D'**とラベル付けされており、
お客様にはアルファベット順でカメラを接続することをお勧めします。
その場合、ビデオデバイスはJetson Linux上で対応する**'/dev/video0'**、**'/dev/video1'**、**'/dev/video2'**、**'/dev/video3'**としてマップされます。

ただし、カメラが1台のみ接続されている場合は、'video0'のみが表示されます。

次に、Jetson IOでデバイスツリーオーバーレイを設定してデバイスを有効にします（設定して再起動するまで**'/dev'**フォルダ下にビデオノードは表示されません）。

### Jetson IO の設定

以下のコマンドでJetsonIOを起動します：

``` sh
sudo /opt/nvidia/jetson-io/jetson-io.py
```

### ビデオ情報の設定

`v4l-utils`をインストールしていない場合：

``` sh
sudo apt-get update 
sudo apt-get install v4l-utils -y
```

+ フレーム同期

``` sh
v4l2-ctl -d /dev/video0 --set-ctrl=trig_mode=1
```

:::note
**'trig_mode=0'**に切り替えてフレーム同期を無効にできます。または複数のカメラが接続されている場合は**'trig_mode=1'**を使用してフレーム同期を有効にします。
:::


+ ビデオフォーマット

``` sh
v4l2-ctl -V --set-fmt-video=width=1920,height=1536 -c sensor_mode=0  -d /dev/video0
```

:::note
使用しているカメラによって、ここで事前定義された異なる'sensor_mode'がある場合があります。主に：

|センサーモード|フレームタイプ|
|   :----:  | :----:   |
|0|YUYV8_1X16/1920x1536|
|1|YUYV8_1X16/1920x1080|
|2|YUYV8_1X16/3840x2160|



:::

### Gst Launch

`gst-launch-1.0`でビューアーを起動します：

``` sh
gst-launch-1.0 v4l2src device=/dev/video0 ! \
    video/x-raw,format=YUY2,width=1920,height=1080,framerate=30/1 ! \
    videoconvert ! videoscale ! xvimagesink
```
