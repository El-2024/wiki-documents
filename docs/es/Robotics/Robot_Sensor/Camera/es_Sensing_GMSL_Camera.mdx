## Primeros Pasos

Usaremos reComputer Robotics J4012 para la demostración, lo que podrías necesitar

+ Cámara GMSL2 SENSING
+ [Un cable Mini-Fakra](https://www.seeedstudio.com/Mini-fakra-Coaxial-Cable-4-in-1-0-5m-Female-to-Female-p-6484.html)
+ [Placa de extensión GMSL2 de reComputer Robotics](https://www.seeedstudio.com/reComputer-Robotics-GMSL-board-p-6511.html)


### Configuración del Dispositivo

Conecte el cable Mini Fakra con la cámara, había 4 canales en reComputer Robotics J30/J40, usamos la cámara `SG3S-ISX031C-GMSL2-F` y está conectada al canal **'1'** (letra de serigrafía **'B'**).

Estos 4 canales están etiquetados como **'A'**, **'B'**, **'C'** y **'D'** en el dispositivo,
recomendamos a nuestros clientes conectar la cámara con el orden alfabético dado,
en cuyo caso sus dispositivos de video se mapearán como **'/dev/video0'**, **'/dev/video1'**, **'/dev/video2'**, **'/dev/video3'** en su Jetson Linux correspondiente.

Pero si solo tiene una cámara conectada, solo se mostrará 'video0'.

A continuación, vamos a configurar la superposición del árbol de dispositivos con Jetson IO para habilitar nuestro dispositivo (no verá nodos de video en la carpeta **'/dev'** hasta que lo configure y reinicie).

### Configurar Jetson IO

Lance JetsonIO con

``` sh
sudo /opt/nvidia/jetson-io/jetson-io.py
```

### Configurar Información de Video

En caso de que no haya instalado `v4l-utils`

``` sh
sudo apt-get update 
sudo apt-get install v4l-utils -y
```

+ Sincronización de Fotogramas

``` sh
v4l2-ctl -d /dev/video0 --set-ctrl=trig_mode=1
```

:::note
Puede alternar **'trig_mode=0'** para deshabilitar la sincronización de fotogramas. O usar **'trig_mode=1'** para habilitar la sincronización de fotogramas cuando tenga múltiples cámaras conectadas.
:::


+ Formato de Video

``` sh
v4l2-ctl -V --set-fmt-video=width=1920,height=1536 -c sensor_mode=0  -d /dev/video0
```

:::note
Dependiendo de qué cámara esté usando, podría haber diferentes 'sensor_mode' como se predefine aquí, principalmente:

|Modo del Sensor|Tipo de Fotograma|
|   :----:  | :----:   |
|0|YUYV8_1X16/1920x1536|
|1|YUYV8_1X16/1920x1080|
|2|YUYV8_1X16/3840x2160|


:::

### Lanzamiento Gst

Lance el visor con `gst-launch-1.0`

``` sh
gst-launch-1.0 v4l2src device=/dev/video0 ! \
    video/x-raw,format=YUY2,width=1920,height=1080,framerate=30/1 ! \
    videoconvert ! videoscale ! xvimagesink
```
