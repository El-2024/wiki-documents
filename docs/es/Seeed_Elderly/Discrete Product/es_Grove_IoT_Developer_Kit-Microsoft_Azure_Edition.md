Grove IoT Developer Kit – Microsoft Azure Edition contiene un **módulo Intel® Edison**, una **placa Intel® Edison para Arduino**, un **Grove Base Shield**, un conjunto de **sensores Grove** y **actuadores** con puertos Grove integrados para prototipado rápido. Simplemente conecta los módulos y estarás listo para crear.

## Intel® Edison Para Arduino

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Intel_Edison_for_Arduino_Microsoft_Azure_Edition_1.jpg)

El módulo de cómputo Intel® Edison está diseñado para makers expertos, emprendedores y algunas aplicaciones IoT industriales, proporciona facilidad de desarrollo para una gama de proyectos de prototipado o empresas comerciales cuando el rendimiento importa. Con una placa de expansión que se interfaz con una amplia gama de shields de Arduino, Intel® Edison para Arduino permite prototipado rápido y fácil con hardware de código abierto y un entorno de desarrollo de software.

## Características

- Una combinación del poder de Intel® Edison y el rico sistema Grove

- Prototipado rápido para proyectos IoT

- Soporta desarrollo con Arduino* y C/C++, seguido por Node.JS, Python, RTOS, y soporte de Programación Visual en el futuro cercano

- Incluye un marco de conectividad dispositivo-a-dispositivo y dispositivo-a-nube para habilitar comunicación entre dispositivos y un servicio de análisis de series temporales basado en la nube y multi-inquilino.

- Tiene un conector de tarjeta SD, micro USB o conector USB host estándar tipo A (vía interruptor mecánico), dispositivo Micro USB, 6 entradas analógicas, y 20 pines de entrada/salida digital, 1x UART, 1x I2C, y 1x cabecera ICSP de 6 pines (SPI) Jack de alimentación con entrada DC de 7V-15V.

## Primeros Pasos con la Placa Intel® Edison

Antes de trabajar con cualquier módulo Grove, primero comencemos con la Placa Intel® Edison. Puedes seguir las instrucciones detalladas en **[el sitio oficial de Intel® Edison](https://software.intel.com/en-us/iot/library/edison-getting-started)** para:

### Paso 1: Ensambla tu Placa

Instala el módulo de cómputo, conecta las patas y enchufa tus cables.

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Intel_edison_assemble.png)

### Paso 2: Elige tu Sistema Operativo

Necesitarás una [**conexión de terminal serie**](https://software.intel.com/en-us/get-started-edison-windows-step3) a tu placa para flashear el firmware

### Paso 4: Conectar tu Placa a Internet

¿Cómo puedes construir un dispositivo IoT sin la "I"? Vamos a [**conectar tu placa a la red WiFi**](https://software.intel.com/en-us/get-started-edison-windows-step4).

### Paso 5: Pruébalo con Arduino IDE

Como ya hemos instalado el entorno de desarrollo incluyendo Arduino IDE en el Paso 2 (si no, puedes descargar el último Arduino IDE **[aquí](https://www.arduino.cc/en/Main/Software)**. ) ahora como tu primera práctica, vamos a intentar [**hacer parpadear un LED en la Placa Intel Edison con Arduino IDE**](https://software.intel.com/en-us/get-started-arduino-blink).

## Ejecutando el Ejemplo

1. Abre el sitio web: [Grove_Indoor_Environment_Demo](https://github.com/Seeed-Studio/Grove_Indoor_Environment_Demo) para descargar todo el proyecto.

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Indoor_Kit_Github_Demo.png)

2. Haz clic en **Tools &gt; Serial Port** y selecciona el Com # al que está conectado el Intel Edison

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Import_Indoor_Kit_Demo.png)

3. Haz clic en Sketch&gt;Import Library…&gt;Add Library e importa la biblioteca descargada en el **paso 1**

4. Haz clic en **File&gt;Examples&gt; Grove_Indoor_Environment_Demo** y selecciona la demostración. Haz clic en el icono **upload**

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Upload_Indoor_Kit_Demo.png)

5. Abre **Serial Monitor**, imprimirá la información de los sensores:

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Indoor_Kit_Serial_Monitor.png)

6. Gira el Encoder para verificar el valor del sensor en el LCD.

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Indoor_Kit_Rotate_Encoder.png)

7. En el **"Send TextBox"**, puedes ingresar el siguiente comando para operar los sensores y actuadores:

<table>
<tr>
<th>Ejemplo</th>
<th>Descripción</th>
</tr>
<tr>
<td width="300px">_set temp&gt;40, relay=1_</td>
<td width="500px">si la temperatura es mayor a 40℃, el relé se abre.</td>
</tr>
<tr>
<td width="300px">_set temp&gt;40, sleep=1_</td>
<td width="500px">si la temperatura es &gt;40℃, no hacer nada.</td>
</tr>
<tr>
<td width="300px">_set humi&gt;60, buzzer=1_</td>
<td width="500px">si la humedad es &gt;60%, el buzzer suena.</td>
</tr>
<tr>
<td width="300px">_set light&gt;600, servo=90_</td>
<td width="500px">si la intensidad de luz es &gt;600, el servo gira 90°.</td>
</tr>
<tr>
<td width="300px">_set uv&gt;80, relay=0_</td>
<td width="500px">si la intensidad UV es &gt;80, el relé se cierra.</td>
</tr>
<tr>
<td width="300px">_set pir=1, buzzer=1_</td>
<td width="500px">si se detectan personas, el buzzer suena.</td>
</tr>
<tr>
<td width="300px">_set ms&gt;40, relay=1_</td>
<td width="500px">si la humedad es &gt;40, el relé se abre.</td>
</tr>
<tr>
<td width="300px">_set ssid=name, psw=password_</td>
<td width="500px">establece el SSID y contraseña del wifi. Puedes abrir un navegador web e ir a la dirección IP mostrada en el Serial Monitor o LCD. El puerto predeterminado es 88. El puerto predeterminado es 88. Por ejemplo: 192.168.1.101:88</td>
</tr>
</table>

Nota:

- El comando debe terminar con 'n', por lo que **"NewLine"** (en el Monitor Serie) debe estar seleccionado.

- Un actuador solo puede ser controlado por un sensor. Si un sensor A quiere controlar un actuador (que ya está siendo controlado por el sensor B), el sensor B debe configurarse en modo sleep.

![](https://files.seeedstudio.com/wiki/Grove_IoT_Developer_Kit-Microsoft_Azure_Edition/img/Indoor_Kit_command.png)

8. Conexión WiFi. Abre el Monitor Serie y configura tu ssid y contraseña (como se muestra a continuación). Verifica la IP local en la LCD o en el Monitor Serie. En un dispositivo conectado a la misma red, abre un navegador web y ve a la dirección IP anterior, podrás ver el valor del sensor.

_**Nota: Al visitar el servidor web, se debe agregar un número de puerto (88), como por ejemplo: 172.20.10.2:88.**_
